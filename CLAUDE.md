## Commands

```bash
npm run dev          # Local dev server at localhost:4321
npm run build        # Runs scripts/fetch-hero.mjs then astro build
npm run preview      # Preview production build locally
```

The build command is intentionally `node scripts/fetch-hero.mjs && astro build`. The pre-build script **must** run before Astro — it fetches Flickr data and writes `src/data/hero.json` and `src/data/photos.json`, which are gitignored and read by pages at prerender time.

## Architecture

**Static Astro 6 site** deployed to Cloudflare Pages. Output mode is `static` with the Cloudflare adapter.

### Content

Two Astro content collections (defined in `src/content.config.ts`):
- `src/content/journal/*.md` — ~247 blog posts with `title`, `date`, `excerpt`, `categories`, `tags`, `draft` fields
- `src/content/podcast/*.md` — podcast episodes with `audioUrl`, `duration`, `fileSize`, `episodeNumber`, `description`, `artwork` fields

### Photo / Hero pipeline

Flickr photos are fetched by `scripts/fetch-hero.mjs` **before** the Astro build runs. This is the only approach that works — env vars are unavailable inside Vite's prerender context (neither `import.meta.env` nor `astro:env/server` work reliably for Cloudflare build-time prerendering).

- `scripts/fetch-hero.mjs` reads `process.env.FLICKR_*`, writes `src/data/hero.json` (random single photo) and `src/data/photos.json` (full gallery array)
- `src/pages/index.astro` and `src/pages/photos.astro` import these JSON files via dynamic `import()`
- In Cloudflare Pages, `FLICKR_API_KEY`, `FLICKR_USER_ID`, and `FLICKR_ALBUM_ID` must be set as **Build variables** (not Runtime variables)

### CMS

Keystatic is integrated (`@keystatic/astro`) for managing journal and podcast content. Currently configured in local mode. GitHub OAuth setup is pending — will require `KEYSTATIC_GITHUB_CLIENT_ID`, `KEYSTATIC_GITHUB_CLIENT_SECRET`, and `KEYSTATIC_SECRET` env vars, and switching `keystatic.config.ts` to `kind: 'github'`.

### Podcast RSS

The iTunes-compatible RSS feed is at `/podcast/rss.xml` (generated by `src/pages/podcast/rss.xml.ts`). Audio files are hosted on Cloudflare R2 at `pub-8e3685a16ec94aae82a07d0f52e4c128.r2.dev`.
